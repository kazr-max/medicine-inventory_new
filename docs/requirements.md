# 薬在庫管理アプリ 要件定義書

## 1. 概要
- **目的**: 自宅の薬の在庫を管理し、残数が少なくなったらLINEで通知するWebアプリ
- **利用者**: 家族（複数ユーザーで利用）
- **プラットフォーム**: レスポンシブ対応Webアプリ（PC・スマホ両対応）

## 2. 技術スタック
| 層 | 技術 |
|---|---|
| フロントエンド | React（TypeScript） |
| バックエンド | Java / Spring Boot |
| データベース | PostgreSQL |
| 通知 | LINE Messaging API |

## 3. ユーザー管理
- 家族間で共有利用するため、ユーザーごとに薬を管理する
- ユーザー認証（ログイン）は不要。ユーザー選択（切り替え）で管理する
- ユーザー情報は名前のみ保持する
- ユーザーの新規登録が可能（ヘッダーの「＋」ボタンから登録）

## 4. 管理する薬の情報（基本情報）
| 項目 | 必須 | 説明 |
|---|---|---|
| 薬名 | YES | 薬の名前 |
| 種別 | YES | 常用（REGULAR）または頓服（AS_NEEDED）。デフォルトは常用 |
| 数量 | YES | 現在の在庫数 |
| 使用期限 | NO | 薬の使用期限（任意） |
| 1日の服用数 | NO | 常用薬の場合、1日に服用する数。自動減算に使用 |
| アラート閾値 | YES | この数以下になったらLINE通知 |

## 5. 機能一覧

| # | 機能 | 説明 |
|---|---|---|
| F1 | 薬の登録 | フォームから手動で薬情報を入力・登録 |
| F2 | 薬の編集・削除 | 登録済みの薬情報を編集・削除 |
| F3 | 在庫一覧表示 | 登録した薬の一覧をリスト表示 |
| F4 | 検索 | 薬名で検索・フィルタリング |
| F5 | 在庫数の増減 | 在庫数をボタン等で簡単に増減 |
| F6 | 在庫数アラート（LINE通知） | 在庫数が閾値以下になった時にLINEで即時通知 |
| F7 | ユーザー登録 | 新規ユーザーをアプリから登録 |
| F8 | 常用薬の自動減算 | 毎朝8時に常用薬の在庫を1日の服用数分自動で減算。閾値を下回った場合はLINE通知 |

## 6. 画面構成

### 6.1 一覧画面（トップ画面）
- ヘッダー: アプリ名「薬在庫管理」、ユーザー切り替え、ユーザー追加ボタンを表示
- ユーザー切り替え: 選択中のユーザーを切り替える（選択したユーザーの薬のみ表示）
- ユーザー追加: 「＋」ボタンからユーザー登録モーダルを開く（F7）
- 検索バー: 薬名で検索・フィルタリング（F4）
- 在庫一覧: 選択中のユーザーの薬をカード形式で一覧表示（F3）
  - 各カードに表示する情報: 薬名、種別バッジ（常用薬の場合）、使用期限（未設定時は「-」）、アラート閾値、1日の服用数（常用薬の場合）、在庫数
  - 在庫数の増減ボタン（＋/－）をカード内に配置（F5）
  - 編集・削除ボタンをカード内に配置（F2）
- 在庫状態の段階表示（カード左ボーダーの色で区別）:
  - 正常（緑）: 在庫数がアラート閾値を上回っている
  - 残少（橙）: 在庫数がアラート閾値以下
  - 要補充（赤）: 在庫数がアラート閾値を大きく下回っている
- FABボタン（右下固定）: 新規登録モーダルを開く（F1）

### 6.2 登録モーダル（F1）
- モーダル形式で一覧画面の上にオーバーレイ表示
- 入力項目: 薬名（必須）、種別（常用/頓服）、数量（必須）、アラート閾値（必須）、1日の服用数（常用時のみ表示）、使用期限（任意）
- アクション: 「登録する」「キャンセル」

### 6.3 編集モーダル（F2）
- モーダル形式で一覧画面の上にオーバーレイ表示
- 入力項目: 登録モーダルと同一（既存値がプリセット）
- アクション: 「更新する」「キャンセル」

### 6.4 ユーザー登録モーダル（F7）
- モーダル形式で一覧画面の上にオーバーレイ表示
- 入力項目: ユーザー名（必須）
- アクション: 「登録する」「キャンセル」
- 登録後、新規ユーザーが自動選択される

### 6.5 削除確認モーダル（F2）
- モーダル形式で確認ダイアログを表示
- 対象の薬名を表示し、削除の最終確認を行う
- アクション: 「削除する」「キャンセル」

### 6.6 LINE通知（F6 / F8）
- 在庫数を減らした際に閾値以下になった場合、即時にLINE Messaging APIで通知
- 常用薬の自動減算（毎朝8時）で閾値を下回った場合もLINE通知
- 通知メッセージ内容: 薬名、残数、閾値

## 7. 非機能要件
- ユーザー認証: なし（ユーザー切り替えのみ）
- レスポンシブデザイン対応（PC・スマホ両対応）
